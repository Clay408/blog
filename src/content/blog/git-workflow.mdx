# Git 工作流最佳实践

良好的 Git 工作流能提高团队协作效率。

## 分支策略

### Git Flow

```
main (production)
  ↑
develop (development)
  ↑
feature/* (功能分支)
hotfix/* (紧急修复)
release/* (发布准备)
```

### Trunk-Based Development

- 所有开发者直接在主分支开发
- 使用功能开关控制新功能
- 频繁集成

## 常用命令

```bash
# 交互式 rebase
git rebase -i HEAD~3

# 修改最后一次提交
git commit --amend

# 暂存未提交的更改
git stash

# 清理未跟踪的文件
git clean -fd
```

## Commit 规范

```
feat: 添加新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具相关
```

## 代码审查清单

1. 代码是否符合团队规范
2. 逻辑是否正确
3. 是否有测试覆盖
4. 性能影响
5. 安全性考虑
